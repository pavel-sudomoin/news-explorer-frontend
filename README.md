## Описание

Проект представляет собой интерактивную страницу, с помощью которой можно искать новости по запросу и сохранять их в личном кабинете.

Проект состоит из двух страниц: главной страницы, с помощью которой можно зарегистрироваться/залогиниться, а также найти требуемые новости и страницы с сохранёнными новостями для залогиненного пользователя.

На данном этапе разработана только вёрстка проекта. Представленный в проекте JS-код предназначен исключительно для удобства тестирования вёрстки.

На последующем этапе будет реализован весь функционал проекта с помощью JS-кода.

Проект разработан в учебных целях как дипломная работа для **Yandex.Praktikum**.

## Используемые технологии

При разработке использован HTML / CSS / JS (ES6).

При сборке использован node.js и webpack с плагинами.

## Тестирование вёрстки

Главная страница загружается в варианте с залогиненным пользователем.

С главной страницы можно перейти на страницу со статьями по ссылке "Сохранённые статьи" в Хидере.

### Реализованный функционал

Исключительно с целью удобства тестирования был реализован следующий функционал:

* Работа Попапов и Хидера на Главной Странице
  - При нажатии в Хидере на Кнопку Выхода (с именем пользователя) меняется Хидер
  - При нажатии в Хидере на Кнопку "Авторизоваться" возникает Попап Логина
  - В Попапе Логина при нажатии на "Зарегистрироваться" возникает Попап Регистрации
  - Все поля Попапов валидируются, при некорректном вводе возникают соответствующие ошибки
  - При прохождении валидации становится активной Кнопка Отправки Формы соответствующего Попапа
  - При нажатии на Кнопку Войти Попап Логина закрывается и меняется Хидер проекта
  - При нажатии на Кнопку Зарегистрироваться Попап Регистрации закрывается и открывается Попап сообщения об успешной регистрации

* Работа Мобильного Меню на всех страницах
  - При нажатии на Бургер-Меню открывается/закрывается Мобильное Меню

### Описание состояний блоков

#### Хидер

Хидер с предложением Зарегистрироваться: `<header class="header">`

Хидер с зарегистрированным Пользователем: `<header class="header header_login">`

Хидер с открытым Мобильным Меню: `<header class="header header_menu-open">`

#### Попап (только для Главной Страницы)

1) Отображение Попапов:

Попапы добавляются в контейнер `<div class="popup">` с помощью JS.

При открытии попапа контейнеру ставится следующий класс `<div class="popup popup_open">`

2) Отображение Ошибок в Форме Попапа:

Для отображения ошибки элементу с классом *popup__error* добавляется класс *popup__error_open*

`<span class="popup__error popup__error_type_email">Неправильный формат email</span>` *//ошибка не отображается*

`<span class="popup__error popup__error_type_email popup__error_open">Неправильный формат email</span>` *//ошибка отображается*

3) Изменение состояния Кнопок

Для активирования Кнопки необходимо удалить у неё класс *popup__button_disabled* и убрать атрибут *disabled*

`<button type="submit" class="button popup__button popup__button_disabled" disabled="disabled">Войти</button>` *//Кнопка неактивна*

`<button type="submit" class="button popup__button">Войти</button>` *//Кнопка активна*

#### Блок search (только для Главной Страницы)

По аналогии с Попапами, для отображения ошибки с формой поиска необходимо элементу с классом *search__error* добавить класс *search__error_open*

#### Блок results

1) Блок Результатов Поиска

Для отображения всего блока Результатов Поиска необходимо добавить этому блоку класс *results_open*

`<section class="results results_open">` *//Блок результатов поиска отображается*

2) Различные состояния результатов поиска

Для отключения тех или иных блоков внутри блока Результатов Поиска, необходимо добавить им класс *results__disabled*

`<div class="results__load results__disabled">` *//Скрыли блок с прелоудером*

`<div class="results__not-found results__disabled">` *//Скрыли блок с сообщением о том, что ничего не найдено*

`<div class="results__found results__disabled">` *//Скрыли блок с найденными статьями*

Аналогично и с кнопкой "Показать ещё":

`<button type="button" class="button results__button results__disabled">Показать еще</button>` *//Скрыли кнопку "Показать ещё"*

3) Отметка о сохранении Статьи (**только для Главной Страницы**)

Если статья не сохранена, то у её кнопки (иконка в верхнем правом углу карточки со статьёй) должен быть класс *article__control_save_unmarked*

При сохранении статьи необходимо заменить этот класс на *article__control_save_marked*

`<button class="article__control article__control_save_unmarked article-legend"></button>` *//Статья не сохранена, иконка не отмечена*

`<button class="article__control article__control_save_marked article-legend"></button>` *//Статья сохранена, иконка отмечена*

4) Подсказка о необходимости регистрации (**только для Главной Страницы**)

По-умолчанию элемент-подсказка о необходимости регистрации для сохранения статей отображается.

Если Пользователь зарегистрировался, то у каждого элемента-подсказки необходимо добавить класс *article__prompt_disabled*

`<p class="article__prompt article-legend">Войдите, чтобы сохранять статьи</p>` *//Пользователь не зарегистрирован, подсказка будет отображена*

`<p class="article__prompt article__prompt_disabled article-legend">Войдите, чтобы сохранять статьи</p>` *//Пользователь зарегистрирован, подсказка не будет отображена*

## Установка

### Собранная версия проекта

Текущая собранная версия проекта (с транспилированным и минифицированным кодом) находится в ветке gh-pages.

Для установки необходимо скопировать файлы из ветки gh-pages к себе в директорию.

### Сборка из исходных файлов

Конфигурационные файлы находятся в корне проекта.

Исходные файлы проекта (HTML, CSS, JS, шрифты и картинки) находятся в папке */src*.

Для сборки необходимо иметь установленный node.js.

Для сборки требуется:

* Cкопировать все файлы из ветки *level-2* к себе в директорию
* Запустить консоль и перейти в папку с проектом
* Командой *npm i* установить все потребные плагины и пакеты

После этого станут доступны три режима сборки:

* режим *Разработки* (команда *npm run dev*)
  - проект будет собран без минификации css и js
  - будет автоматически запущен сервер и загружен собранный проект
  - при изменении файлов в папке */src* страница будет перегружаться автоматически
* режим *Сборки* (команда *npm run build*)
  - проект будет собран с минификацией css и js
  - результат сборки будет помещён в папку */dist* в корне проекта
* режим *Публикации* (команда *npm run deploy*)
  - проект будет собран по аналогии с режимом *Сборка*
  - затем проект будет размещён на двух серверах - github и на облачном сервере от *Яндекс.Облако*
